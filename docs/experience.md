# What I Did

This case study is based on the kind of work I performed as part of an Azure Landing Zone program in a professional environment.

My involvement was not limited to day-to-day operations. I contributed across design, implementation, platform improvement, and long-term onboarding support.

## Core Areas Of Involvement

- contributed to the overall infrastructure-as-code structure and platform flow
- helped improve and fix design decisions as the platform matured
- developed and maintained core Terraform modules for foundational Azure infrastructure
- supported governance-aware platform implementation across multiple environments
- participated in onboarding application teams onto standardized platform modules

## Foundation Work

My work included direct contribution to reusable modules for:

- management groups
- subscriptions
- resource groups
- virtual networks
- subnets
- network security groups
- user-defined routes
- hub connectivity and peering

## Delivery And Tooling

The platform engineering workflow used tools and practices such as:

- Terraform Enterprise
- GitHub
- GitHub Actions
- `terraform-docs`
- Azure native services and governance controls

## Operating Model Exposure

I worked across several important platform concerns, including:

- workspace and environment separation
- topology-aware deployment patterns
- service principal concepts
- RBAC participation
- Azure Policy participation
- module-driven onboarding for downstream applications

## Scale Theme

One of the strongest signals from this experience is scale. The platform was greenfield, and over time many workloads were onboarded through reusable service modules and shared platform standards.

This repository does not reproduce that environment directly, but it is meant to reflect the engineering style and platform mindset behind that work.
